================================================================================
  FIX: "Not Found" / Link not working (404 on Vercel)
================================================================================

If you see 404 when opening /soulplace/login, /soulplace/menu, /soulplace/links,
or the infinity-symbol Vercel 404 page:

  STEP 1 – SET ROOT DIRECTORY (required)
  -------------------------------------
  1. Open: https://vercel.com/dashboard
  2. Click your project (e.g. soulplace-help-desk)
  3. Settings → General
  4. "Root Directory" → Edit → set to:  soulplace_help_desk
     (The folder that contains app.py, api/, templates/, config.json)
  5. Save

  STEP 2 – PUSH CODE AND REDEPLOY
  -------------------------------
  6. Push the latest code (with api/index.py and vercel.json rewrites) to GitHub.
  7. Deployments → "..." on latest → Redeploy (or push triggers a new deploy).
  8. Wait for the deploy to finish, then try the links again.

  WHY THIS HAPPENS
  ----------------
  Vercel must build from the folder where app.py and api/index.py live.
  If Root Directory is wrong, the Flask app is not run and every path returns 404.
  The project now includes api/index.py and rewrites so all routes go to Flask.

  CORRECT URLS (after fix)
  -----------------------
  Login:   https://soulplace-help-desk.vercel.app/soulplace/login
  Tables:  https://soulplace-help-desk.vercel.app/soulplace/tables
  Menu:    https://soulplace-help-desk.vercel.app/soulplace/menu
  Links:   https://soulplace-help-desk.vercel.app/soulplace/links

  IF YOUR REPO IS ONLY THE soulplace_help_desk FOLDER
  --------------------------------------------------
  If you pushed only the contents of soulplace_help_desk (so app.py is at
  the repo root), then leave Root Directory EMPTY and redeploy.
